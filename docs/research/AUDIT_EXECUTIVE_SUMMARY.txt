CRITICAL FINDINGS (🔴 MUST FIX BEFORE SHIP)
================================================================================

1. RLS POLICIES COMPLETELY OPEN
   File: /docs/SPRINT_13_MIGRATION.sql (lines 176-200)
   Problem: All INSERT/UPDATE policies use "WITH CHECK (true)"
   Impact: ANY USER can modify ANY OTHER USER'S DMS
   Risk: Attacker can cancel journalist's protection, suppress alarms
   
   Fix Required:
   - Change RLS to owner_fingerprint checks
   - Set INSERT/UPDATE to "USING (false)" for anon clients
   - Only service role API routes can write

2. FINGERPRINT AUTHENTICATION MISSING
   Files: /api/collective-dms/route.ts, /api/dms/route.ts
   Problem: API accepts fingerprint from request body without verification
   Impact: Attacker can impersonate any user
   Risk: Can pause/cancel victim's DMS from remote
   
   Fix Required:
   - Get fingerprint from secure cookie/header
   - Remove from JSON body parameters
   - Use JWT or session tokens

3. RECOVERY KEY STORAGE INSECURE
   File: /lib/deadManSwitch.ts (line 133)
   Problem: AES-256 key returned to client, no backup mechanism
   Impact: Lost key = lost encrypted documents (PERMANENT DATA LOSS)
   Risk: Gazeteci's life work disappears if key is lost
   
   Fix Required:
   - Use Shamir's Secret Sharing to split recovery key
   - Distribute key shards to guarantors
   - Document recovery procedure

================================================================================
HIGH-PRIORITY FINDINGS (🟡 FIX IN NEXT SPRINT)
================================================================================

4. SHAMIR IMPLEMENTATION NOT TESTED
   File: /lib/shamir.ts
   Problem: Complex crypto with zero unit tests
   Impact: Silent failures, unrecoverable documents
   Risk: Only detected when journalist needs recovery
   
   Fix: Add comprehensive test suite (M-of-N combinations)

5. PDF METADATA STRIPPING INCOMPLETE
   File: /lib/crypto.ts (lines 354-361)
   Problem: Only renames file, doesn't strip internal PDF metadata
   Impact: Author name, creation date, software still visible
   Risk: Journalist identified via PDF properties
   
   Fix: Use pdf-lib or server-side processing to strip metadata

6. RATE LIMITING IN-MEMORY ONLY
   File: /lib/rateLimit.ts
   Problem: Map resets on server restart, no clustering support
   Impact: DoS protection bypassed on deployment
   Risk: Attacker floods API during rolling update
   
   Fix: Use Redis/Memcached for distributed rate limiting

7. CRON JOB NO AUTHENTICATION
   Files: /api/dms/cron/route.ts, /api/collective-dms/cron/route.ts
   Problem: Any client can trigger DMS manually
   Impact: Premature document release
   Risk: Journalists get blown away during mission planning
   
   Fix: Add CRON_SECRET header validation

================================================================================
MEDIUM FINDINGS (🟠 ADDRESS IN 3 MONTHS)
================================================================================

8. SUPABASE ANON KEY OVER-PRIVILEGED
   File: /lib/supabaseClient.ts
   Problem: Client anon key has too many permissions
   Impact: Direct database queries bypass API validation
   Risk: RLS is only defense; if broken = data breach
   
   Fix: Revoke unnecessary permissions, whitelist SELECT only

9. INPUT VALIDATION INSUFFICIENT
   File: /api/evidence/submit/route.ts
   Problem: No string length, enum validation
   Impact: XSS, memory exhaustion
   Risk: Slow DoS via huge strings
   
   Fix: Add MAX_LENGTH, enum validation for evidence_type
