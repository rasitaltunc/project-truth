CROWD-SOURCED LEGAL DOCUMENT VERIFICATION - QUICK REFERENCE CARD
================================================================================

THE CORE INSIGHT:
  Effective verification tasks are MICRO-NARRATIVES (story context), not
  micro-tasks (isolated facts).

  BAD:   "Verify: Is 'John Smith' in document?" [Yes/No/Unsure]
         → 70% quit rate, random answers

  GOOD:  "MAXWELL NETWORK: You've verified 15 people. Now verify John Smith
         (officer, page 5) against database John Smith (DOB 1/15/1975, NY)"
         → 15% quit rate, 87% accuracy

================================================================================
7 CORE PRINCIPLES (Priority Order)
================================================================================

1. CONTEXT OVER ISOLATION
   - Show WHY this verification matters
   - Show network progress (X people verified so far)
   - Show case narrative

2. BINARY CLARITY
   - Each task: ONE decision point (not "verify all 5 facts")
   - Use conditional branching (question 1 → if answer matches, ask question 2)

3. TEMPORAL CONSISTENCY
   - Build across sessions (Session 1→2→3 reveals patterns)
   - Track user progress visibly

4. CONSENSUS BUILT IN
   - 3+ people see same task
   - Disagreement = escalation, not averaging

5. IMMEDIATE FEEDBACK
   - Show within seconds: "You matched with 2 others" ✓
   - Show accuracy: "Your accuracy: 18/20 (90%)"
   - Show impact: "You added 1 person to network"

6. NO FINANCIAL INCENTIVES
   - Money kills quality (proven in MTurk studies)
   - Use: Reputation, recognition, tier progression, leaderboards

7. EDUCATIONAL ARC
   - Users LEARN while verifying
   - Teach tools, context, why each person matters

4 TASK TEMPLATES (Ready to Code)
================================================================================

TEMPLATE 1: EXISTENCE VERIFICATION (Tier 1, Easiest)
  Duration:    20 seconds
  Task:        "Does [name] appear in this document excerpt?"
  Accuracy:    92%
  Quit rate:   <5%
  Use for:     Onboarding, basic entity extraction
  Example:     "Does 'Ghislaine Maxwell' appear in this text?" [YES/NO/UNCLEAR]

TEMPLATE 2: DETAIL MATCHING (Tier 1-2, Medium)
  Duration:    45 seconds
  Task:        "Do these details match?" (document vs. database)
  Accuracy:    88%
  Quit rate:   8-12%
  Use for:     Verify identity (is this THE John Smith?), detect duplicates
  Example:     Document says "John Smith, DOB 1975, NYC"
               Database says "John Smith, DOB 1975, 123 Main St, NY"
               [SAME PERSON] [DIFFERENT] [UNSURE]

TEMPLATE 3: RELATIONSHIP VERIFICATION (Tier 2+, Hard)
  Duration:    60 seconds
  Task:        "Did these two people know each other?" (with evidence)
  Accuracy:    78%
  Quit rate:   15-20%
  Use for:     Build connection network, classify evidence
  Requires:    Tier 2 unlock (50+ correct previous tasks)
  Example:     Person A: Jeffrey Epstein
               Person B: Ghislaine Maxwell
               Evidence: Email dated March 15, 1991: "Meeting with GM at 3pm"
               Evidence Type: [☐ Communication] [☑ Meeting] [☐ Financial] [☐ Shared Location] [☐ Third-party]
               [DEFINITELY YES] [PROBABLY YES] [PROBABLY NO] [DEFINITELY NO] [NEED MORE INFO]

TEMPLATE 4: EXPERT REVIEW (Tier 3, Dispute Resolution)
  Duration:    3-5 minutes
  Task:        Show when community disagreed; expert decides
  Accuracy:    90%+
  Availability: Tier 3 experts only (100+ correct + 90% calibration + invitation)
  Use for:     Escalated disagreements, audit trail
  Decision:    BINDING (no further appeals)

TEMPLATE 5: FEEDBACK LOOP (After Every Task)
  Duration:    3 seconds
  Shows:       ✓ CORRECT! You matched with 2 others.
               Your accuracy: 18/20 (90%)
               You added 1 person to network. (147 verified so far)
               [NEXT TASK] [VIEW PROFILE] [TAKE BREAK]

================================================================================
ANTI-GAMING MECHANICS (Essential)
================================================================================

HONEYPOTS (Every 10 tasks):
  - Secret known-answer task embedded
  - If user fails: pause, show feedback
  - Catches cheaters, identifies quality workers

RATE LIMITING:
  - Max 30 tasks per session
  - 5-minute break recommended after 30 (cognitive fatigue research)

ASYMMETRIC SCORING:
  - Correct:  +1 point
  - Wrong:    -2 points (penalty > reward, forces honesty)
  - Unsure:   0 points, no penalty

STREAK SYSTEM:
  - 5 correct in a row:  +2 bonus on next task
  - ONE wrong answer:    streak resets to 0
  - Forces consistency, not speed

================================================================================
FAILURE MODES TO AVOID
================================================================================

CAPTCHA TRAP:
  Problem:  Remove all context to "reduce bias"
  Result:   Users quit (no story = no engagement)
  Fix:      Always show 1-2 sentences of case context

CROWD IS WISE:
  Problem:  Average crowd opinion for legal claims
  Result:   Inaccurate, legal liability
  Fix:      Crowd verifies facts; experts judge claims (separate)

GAMIFICATION GRIND:
  Problem:  Points, badges, competitions everywhere
  Result:   Manipulation, toxicity, burnout
  Fix:      Reputation + recognition >> points

SPEED TRAP:
  Problem:  Pay per-task or reward speed
  Result:   People rush; accuracy drops to 60-70%
  Fix:      Fixed session payment; reward accuracy, not speed

EXPERT BOTTLENECK:
  Problem:  Require expert review for everything
  Result:   Slow, doesn't scale
  Fix:      Tier system (crowdsourced facts, expert claims)
